# What I Would Do Next

## Priority 1 - Fundamentally Improve the Experience
- Implement a proper task instructions viewer that is presents the instructions in an ordered manner with helpful interactable examples.
- Add additional flows for other possible users from other different departments within the company, such as allowing the HR person to add the new employee and the tasks assigned to them to the database.
- Add authentication to the application and role based access control, since it's currently not implemented.
- Extend the chatbot to a fully fledged LLM-based agent that could, through natural language interaction, redirect the user to the correct page depending on their query or provide the information by accessing the application's available API endpoints. This feature I consider one of the most flexible in terms of user interaction and actual implementation.

## Priority 2 - Add Significant Value
- Implement caching for API request responses
- Add proper error handling and validation
- Add a complete automated tests suite

## Priority 3 - Nice-to-Have Improvements
- The content management system and configuration file approach I thought of early in development, which would allow content managers to update the application's content through a configuration file.
- Improved UI design and dark theme consistency.
