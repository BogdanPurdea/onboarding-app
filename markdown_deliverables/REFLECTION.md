# Reflection

* **What turned out to be harder than expected?**
  - The planning phase and defining the application ended up one of the harder parts due to the vagueness of the project requirements.
  - The persitance implementation idea ended up being more complex than expected, with the current implementation only allowing recovery of checklist progress through a copied link. This felt like a problem that I did not fully solve and would need to revisit.
* **Which decision would be made differently if started over?**
  - I'd take more time to think about the implementation details, the edge cases and the features that would provide significant value to the user from the beginning.
  - I'd consider implementing a very lightweight authentication feature for covering the persistance feature instead of the current provisional solution.
* **What did I learn about myself as a dev during this project?**
  - I became aware that I initially focus on the happy path, and do not take enough time to consider most possible edge cases and how to handle them gracefully. I do not focus enough on error handling and validation at the time of implementing a new feature.
